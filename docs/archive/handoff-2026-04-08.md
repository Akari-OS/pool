## 🚨 既知の問題・注意点

1. **edition 2021 固定**: Rust 1.85 を避けるため `edition = "2021"`。edition 2024 にはしない
2. **ItemType::Custom(String)**: serde Deserialize 対応で `&'static str` ではなく `String` にしてある
3. **SQLite Connection は `Mutex<Connection>`**: 各 workspace ごとに独立。`Pool` 側で `HashMap<String, Arc<Workspace>>` キャッシュ
4. **FTS5 は trigram**: 日本語部分一致対応。ただし 1-2 文字クエリは空結果
5. **Lint findings の上書き方式**: `akari-pool lint` 実行ごとに既存 unresolved を DELETE して INSERT。resolved_at つきの古いレコードは残る
6. **Wiki は片方向**: Phase 4 時点では Pool → .md のみ。Obsidian で編集しても次回 compile で上書きされる（Phase 5 で双方向を検討）
7. **notes/{uuid}.md**: ファイル名は UUID、タイトルは frontmatter の `title`。Obsidian ではタイトル表示になる
8. **Relation の横断リンク**: `target_workspace` が非 NULL の場合は compile_note の「関連」セクションから除外（Phase 4 v1）
9. **Inconsistency / ConnectionGap 未実装**: Phase 4 Linter は Missing + Orphan の 2 種のみ
10. **API キーはチャット履歴に残さない**: Phase 3 時に平文でチャットに貼ってしまった履歴がある。次セッションで新しいキーを発行して `~/.akari-pool/.env` に貼る（このファイル経由なら安全）。**前回のキー `sk-or-v1-f87ed...` は revoke すること**

## 📝 重要な決定ログ（Phase 2〜4 で確定した方針）

1. **Phase 2**: `Pool` を workspace マネージャに refactor。`Workspace` 構造体が CRUD の責務を持つ。Phase 1 の `Pool::add_item` 等の API は削除（MVP 段階なので破壊変更 OK）
2. **Phase 2**: カレント workspace 解決は `--workspace` > `AKARI_POOL_WORKSPACE` > config.toml > "default" の優先順
3. **Phase 2**: `db.rs` を削除して `pool.rs` + `workspace.rs` に責務分離
4. **Phase 2**: 変換関数は `Display + FromStr` impl に統一（Phase 4 で `RelationType` / `CreatedBy` も追随）
5. **Phase 3**: Analyzer は別 crate（`analyzers/<modality>`）として設計。core に統合しない
6. **Phase 3**: `add_item` は同期のまま、`analyze_item` と `add_item_and_analyze` を async で追加。CLI は `--analyze` フラグで切替
7. **Phase 3**: パーシャル成功方針（分析失敗しても item は保持）
8. **Phase 3**: API キーは `~/.akari-pool/.env` に置く（dotenvy で自動読込、`.gitignore` 済み）
9. **Phase 3**: 標準セットアップは `OPENROUTER_API_KEY` 1 つだけ。ローカル LLM はオプション
10. **Phase 3.5**: 既存 DB は破棄してマイグレーションコード不要（未公開のため）
11. **Phase 3.5**: FTS5 は trigram トークナイザに変更（日本語部分一致のため）
12. **Phase 4**: Wiki Compile は片方向のみ（Pool → .md）。双方向同期は Phase 5
13. **Phase 4**: Linter は trait なしの struct（YAGNI、Phase 4.5 で 4 種揃ったら trait 化検討）
14. **Phase 4**: Lint findings の DB 保存は「既存 unresolved を全削除 → 全 insert」の単純上書き
15. **Phase 4**: notes/{uuid}.md、frontmatter の `title` が Obsidian での表示名
