# Security Policy — AKARI Pool Docs

AKARI Pool ドキュメントコミュニティはセキュリティを真剣に受け止めている。
脆弱性や誤情報を発見したら、責任ある方法で報告してほしい。

---

## 報告の対象

### 報告してほしい事項

- **セキュリティに関わる誤記** — 認証・暗号化・脆弱性対策の説明の誤り
- **不完全な脅威モデル記述** — セキュリティ前提の欠落や矛盾
- **API 仕様の悪用リスク** — 公開ドキュメントに潜む設計上の脆弱性
- **その他の重大な誤情報** — ドキュメント内容に由来するセキュリティインシデント回避

### 報告不要な事項

- 実装の脆弱性 → [`Akari-OS/pool-impl`](https://github.com/Akari-OS/pool-impl) に報告
- アーキテクチャ提案や改善案 → Issue / Discussions で公開討議

---

## 報告方法

### ⚠️ 公開 Issue を立てないこと

セキュリティ関連の情報を公開 Issue に書くと、悪用される可能性がある。
必ず**プライベートな方法**で報告してほしい。

### 報告チャネル

**優先**: 該当リポジトリの Security タブ →「Report a vulnerability」を使う。
GitHub 上で暗号化された報告が送れて、追跡もしやすい。
（例: `https://github.com/Akari-OS/pool/security/advisories/new`）

**代替**: メールで報告 → security@akari-oss.app

---

## 報告に含めてほしい情報

- 影響を受けるドキュメント・セクション
- 誤情報・欠落の具体的な内容
- セキュリティへの潜在的な影響
- 可能であれば修正案

---

## 対応プロセス

1. **受領確認** — 72 時間以内に受領を確認する
2. **初期評価** — 1 週間以内に影響を評価する
3. **修正開発** — ドキュメント修正を準備
4. **リリース** — 修正版を公開
5. **謝辞** — 希望に応じて Contributors に名前を記載

---

## 対応速度の目安

| 重大度 | 例 | 対応スケジュール |
|---|---|---|
| **Critical** | 認証バイパス等の誤記述 | 即日〜3 日 |
| **High** | 暗号化・脆弱性対策の誤り | 1〜2 週間 |
| **Medium** | API 設計の潜在的リスク | 2〜4 週間 |
| **Low** | 表記の曖昧さ | 次回更新時 |

---

## スコープ

本ポリシーは [`Akari-OS/pool`](https://github.com/Akari-OS/pool) リポジトリに適用される。

### スコープ外

- 実装の脆弱性（pool-impl に報告）
- サードパーティドキュメント
- 古い版のドキュメント

---

## 謝辞

責任ある開示をしてくれた報告者には、修正時のコミットメッセージまたは
CHANGELOG に名前を記載する（希望しない場合は匿名）。

**ありがとう**。あなたの報告が AKARI Pool を安全にする。
